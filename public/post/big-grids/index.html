Modeling air quality

When creating a large receptor grid for a statewide air model it’s important to be as efficient as possible when placing receptors. Let’s start with a simple square grid.

How could we increase the map coverage without adding receptors?

Staggered grid

Let’s create a staggered grid by nudging every other row over by one half of the spacing distance. Here’s a small example.

stagger <- data_frame(X = rep(c(seq(0, 7500, 2500), seq(1250, 8750, 2500)), 2),
                      Y = rep(seq(0, 7500, 2500), each = 4))

draw_grid(stagger)

Now the maximum distance from any point on the map is, hmmm… Well that seems a little complicated. Here’s the triangle we want to find the center of.

I think this is going to require some good ol’ Trigonometry.

What do we already know about this triangle?

To start off with, we know the length of the top segment is 2,500. And the length of the line from the bottom point to the middle of the top segment is also 2,500. So that’s something.

draw_triangle()

ArcTan to the rescue

The trig function ArcTan can calculate the unknown angle of a triangle’s corner using the length of the adjacent side and the length of the side across form the corner. If we cut the top segment in half to get 1,250, then we’ll have everything we need to calculate the angle of the top left corner.

draw_grid(stagger)

draw_triangle()

# Top length
text(3180, 5200, "<- 1,250 ->", cex = 0.8, col = "darkorange")

# Vertical line
segments(3750, 2500, 3750, 5000, col = "darkblue", lty = "dashed")

text(4100, 4100, " -- 2,500", cex = 0.8, col = "darkorange")

# Angle
text(2710, 4830, "?", cex = 1.1, col = "darkorange")

The length of the adjacent side is 1,250 and the length of the opposite side is 2,500. So the equation for the angle in degrees is atan(2500 / 1250) * 180 / pi:

Angle in degrees = atan(2500 / 1250) * 180 / pi

Yes!

Now let’s cut this angle in half, creating a new triangle with an edge running from the center of the top segment to the point we are trying to solve for.

Now we can use Tangent to solve for the length of the side opposite the corner angle of 63.44 / 2 = 63.44 / 2 degrees. Side length = tan(half_angle x pi / 180) x 1250.

Side length = tan(half_angle * pi / 180) * 1250

Now all that’s left to do is to subtract this length from the top segment of the original large triangle. So 5,000 - 772.54 = 5000-772.54 and we finally have the Y-coordinate for the center of the big triangle. The X-coordinate is the same as the point below it, 3,750.

New distance

The distance from this point to each of the triangle’s corners is
